Manually booting from GRUB

2026-02-23

Sometimes, (at the fault of my own) GRUB falls over and fails to boot my system. Assuming this is not a catastrophic issue and more so something like mistakenly forgetting a grub-mkconfig after installing a different kernel, or the kernel complaining it does not have the correct root partition passed to its boot parameters, not all hope is lost. Reboot and enter GRUB's command line (pressing C) and poke through the available file systems to find where the kernel and initramfs live: 

grub> ls
(hd0) (hd0,gpt0) (hd0,gpt1) (hd0,gpt2)
grub> ls (hd0,gpt1)/
vmlinuz-6.10 vmlinuz-6.10.old initramfs-6.10 initramfs-6.10.old
grub> root=(hd0,gpt1)
grub> linux /boot/vmlinuz-6.10 root=/dev/sda3
grub> initrd /boot/initramfs-6.10
grub> boot

Here, /dev/sda3 is the partition value passed to the kernel to boot. This is where / lives. However, one may consider omitting GRUB entirely and booting directly from an EFI Stub.
