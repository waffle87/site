Without the physical space or electricity funding to have a home lab setup, I opted for a VPS instead. Despite some perhaps questionable reviews and information online, RackNerd has provided well for my needs. I pay about 25 USD per year for a KVM VPS with 4 GB RAM and 4 virtual CPU cores. While this is a suspiciously low price (what one may pay per month with another provider), they have quite good Black Friday sales. I initially chose them, as one of the requirements for a VPS was the necessary SMTP ports open for a email server. Most providers (AWS, DigitalOcean, etc.) keep these ports strictly closed, as to not be placed on an IP blacklists for spam origination. I will note that my VPS's IP is on a number of blacklists and it has been a work in progress to be able to send email to certain receivers.

There is not much to discuss in terms of intricate system configuration or hardware quirks. (Almost) all services run in Docker containers, configured via Docker Compose. One external network links each container to the Caddy web server container, allowing each container to be configured and run independently: 

services:
  foo:
    networks:
      - caddybridge

networks:
  caddybridge:
    external: true

If a container requires additional images within, like a database, an internal network is used: 

services:
  foo:
    networks:
      - caddybridge
      - foo_dbnet

  db:
    image: mariadb
    container_name: foo_db
    restart: unless-stopped
    environment:
    MYSQL_RANDOM_ROOT_PASSWORD: yes
    MARIADB_DATABASE: foo
    MARIADB_USER: foo_user
    MARIADB_PASSWORD: ${MARIADB_PASSWORD}
    volumes:
      - ./db:/var/lib/mysql
    networks:
      - foo_dbnet
    depends_on:
      - foo

networks:
  caddybridge:
    external: true
  foo_dbnet:
    internal: true

I use MariaDB for any database needs, where possible, for consistency and as it is quite lightweight.
