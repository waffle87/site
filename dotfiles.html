(defcfg
  input  (device-file "/dev/input/by-path/pci-0000:00:14.0-usb-0:5:1.1-event-kbd")
  output (uinput-sink "My KMonad output"
  "/run/current-system/sw/bin/sleep 1 && /run/current-system/sw/bin/setxkbmap -option compose:ralt")
  fallthrough true
  allow-cmd true
)

(defalias
  ctlz (tap-hold 170 z ctl)
  ctldot (tap-hold 170 KeyDot ctl)
  sfa (tap-hold 170 a sft)
  sfcln (tap-hold 170 ; sft)
  altq (tap-hold 170 q alt)
  altp (tap-hold 170 p alt)
  alt1 (tap-hold 170 1 alt)
  alt0 (tap-hold 170 0 alt)
  tgnum (layer-toggle num)
  lt1 (tap-hold 150 spc @tgnum)
  clipst C-S-v
)

unfortunately, laptop keyboards can't be replaced with small ergo keyboards, so kmonad is the next best thing (or my laptop's favorite feature). it's quite full featured considering the scope of the project, with layers, mod-taps, macros, etc. despite the minor lack of documentation, i've been able to recreate my commonly used layout across my QMK compatible keyboards. this is mostly based on a 3x10 layout, with 4 thumb keys. my biggest trouble is not having found a better place for backspace without moving other keys around. normally this is solved with a split spacebar, or combos, sadly one of the only features kmonad is missing. attached is my *.kbd config file which may be useful.
